Avnav Releases

The directory with all releases you will find here.

Hints

The  installation instructions have a detailed description on the different options you have to install AvNav. On this page you will find information about the various releases.
Not with any update there will be new images (although for most). Intermediate updates are either developer versions(daily) or releases.
To install such an intermediate update you need a command line on the pi. From windows you could e.g. use  putty . Just get the address of your pi and connect, use pi, paasword: raspberry.
How to install the packages you will also find in the  installation instructions .
Afterwards you potentially need to adapt the configuration
/home/pi/avnav/data/avnav_server.xml
This could be done like

sudo systemctl stop avnav nano /home/pi/avnav/data/avnav_server.xml # dann Ã¤ndern und speichern sudo systemctl start avnav

If you have installed the AvNav Update plugin you can edit the configuration from this plugin without the need for a command line.

You will find only hints for released versions on this page. For daily builds refer to the commits on GitHub .

20210115 link

  • #73: shift to python3 (the current python2.7 is out of maintenance)
  • plugin API java script extension for featureFormatter
  • decoding of NMEA XDR records
  • formatter for presssure and temperature
  • extension of the java script API - registration of formatters
  • improved handling of formatterParameters in the layout editor
  • NMEA filter for AVNSocketReader,AVNBluetoothReader and AVNUdpReader
  • the avnav history plugin is now available in the package repository and can be installed with
    sudo apt-get install avnav-history-plugin
  • Bugfixes:
    • #97: NMEA checksum in lowercase
    • correct handling of chart dim for night mode
  • Installation Hints
    When installing packages new dependencies are necessary.
    sudo apt-get update
    sudo apt-get upgrade
    If you are using the headless images you need to set the time before installing.
    sudo date -s "2021/01/15 15:00"
    It is important to install the current version for the ocharts-plugin (20210115).
    On Windows a new Installer should be used.
    If you are running an update with the existing installation you need to run it twice! On the first update there is still no installation of the new python version.
