Avnav Releases

The directory with all releases can be found here.

Hints

The  installation instructions have a detailed description of the different options to install AvNav. On this page you will find information about the various releases.
Not every update will be delivered by new images (although available for most). Intermediate updates are either developer versions (daily) or releases.
To install such an intermediate update you need to enter a command line on the pi (or you have the AvNav update plugin installed). From windows you could e.g. use  putty. Just determine the address of your pi and connect, use pi, password: raspberry (or the values you have customized for your image).
Information on how to install the packages you will find in the installation instructions .
Afterwards you potentially need to adapt the configuration
/home/pi/avnav/data/avnav_server.xml
This could be done by

sudo systemctl stop avnav nano /home/pi/avnav/data/avnav_server.xml # enter modifications and save sudo systemctl start avnav

If you have installed the AvNav Update plugin you can edit the configuration file from this plugin without the requirement to enter the command line.

In this documentation you will only find notes for released versions. For daily builds refer to the commits on GitHub .

20250723 link

Main Changes

  • The computation of AIS data has changed to be based on the estimated target position. For details refer to AIS computations.
    This solves #397, #398.
  • The Feature Info handling has been changed. You will now first get a list of objects and can select which one you would like to see the details for. There is also a new "CenterAction" button to "click" on the map center.
  • You can get more flexibility in organizing your widgets on dashboard pages with a new Combined Widget.
  • There are improvements in the layout editor. You can freely drag and drop items between panels - and there is a roll back button.
  • A couple of settings have been removed. Most of them can be applied in the layout editor.
    To support systems with only a basic set of sensors (GPS) an additional default-min layout has been added.
    Removed Setting Replacement
    red/green Angle Wind set this at the wind widgets in the layout editor
    relative motion vectors
    		 set relative motion vector range (nm) to a value != 0
    show clock
    		 include or exclude the clock widget using the layout editor
    show zoom
    		 include or exclude the zoom widget using the layout editor
    show wind
    		 include or exclude the wind widget using the layout editor
    show depth include or exclude the depth widget using the layout editor
    wind knots set the units at the wind widgets (formatter parameters)
    Show Measure Button the measure function is now included in the Feature Info handling
    Always Info on Chart Click now you only have "Feature Info on Click" to disable chart click handling completely. You still can emulate a click at the center (crosshair) with the "CenterAction" button
    If you load a settings file that contains removed settings you will get a warning (but the file still loads).
  • The Edit Route Dialog has been changed. It should be much easier to load/save/modify/rename/create routes. The page to edit route points has been removed and the function is integrated in the Edit Route Dialog.
  • The AIS info page has been replaced by a dialog. You will get this dialog from the Feature List if you selected an AIS target or by clicking on the AisTargetWidget.
  • When you are creating a new User App you can directly upload the necessary icon file from the creation dialog. Additionally you can choose icons that are part of AvNav.
    This resolves #372: allow icon upload directly in user app dialog
  • The handling for missing values (i.e. data that has not been received by NMEA or from SignalK) has been adapted. In the past many widgets or other displays were showing "0" for non existing values. This has been fixed. This resolves #347/#348 handling for missing values
  • Widgets with a formatter that have a "unit" parameter can use this value as the unit to be displayed (by leaving the unit empty in the layout editor)
  • The font handling for items on the map has been improved. There are settings (settings/map) to control some font parameters and overlays also include font parameters in the settings.
  • The UDP reader is able to handle multicast
  • AvNav can decode PGNs 129025/129026 directly from canboat
  • Some internal restructuring (new library versions - especially reactjs 18), restructured dialog handling
