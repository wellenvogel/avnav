Configuration of User Apps

From the settings page you will get here using the button.

A user app is an external or internal HTML page that will be shown at the user app page in an iframe.

All configured user apps will be shown.

Buttons

Icon Name Function

MOB man over board (see main page)
+ AddonConfigPlus add a new configuration

AddonConfigAddOns show the user app page

Cancel back to the previous page

For each configuration the url and an optional title will be shown. Additionally there will be a hint where this configuration has been created (plugin in the picture). If the configuration currently does not work (e.g. missing icon) "invalid" will be shown. By clicking on the configuration you can modify it. Clicking on will move you to the user app page an the selected configuration will be shown there.
When adding or editing a configuration a dialog will be shown.

You can select whether you would like to use an internal or external HTML page (check box), in th epicture an internal page is selected. If you select an external page you need to provide the complete url in the form http(s)://.... If the page is located at the same server like AvNav (like the signalk Web UI) you should replace the hostname with $HOST in your url. AvNav will automatically use the correct ip address in this case.

Internal html pages need to be uploaded beforehand into the user file directory.

For each of the user apps you need an icon file (svg, png, jpeg). The alos have to be uploaded via the Files/Download page either to the user files or to the images category.
After selecting the url and the icon you can provide a title. If you leave the title empty there will be no header at the page.
After saving you can directly test it using the button.

If you used an internal url and later in delete the HTML file on the files/download page the user app configuration will be removed too.
If a used icon file will be deleted the user app configuration will remain but will get invalid. Invalid configurations will not be shown at the user app page.
But you still can change the configuration on this page an make it valid again.

If a user app configuration is store, the server will write back it's avnav_server.xml file. To avoid situations that AvNav cannot start the next time because something went wrong during this writing back there is an avnav_server.xml.ok that is written on every successful start of AvNav.
If AvNav later on is unable to start due to a corrupted avnav_server.xml if will read this fallback file instead.
