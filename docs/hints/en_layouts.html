Layout Adaptation

The displays (widgets) on the navigation page and the dashboard pages are controlled by a configuration json file ("layout").A couple of predefined layouts are included with AvNav. They all have the "system." name prefix. As a user you can define your own layouts and store them (they will have the name prefix "user."). Those user layouts will be stored at a folder "layout" below the BASEDIR (/home/pi/avnav/data on the raspberry). You can directly edit those files, download and upload them - or you can modify the layout using the built in layout editor.

The preferred way of changing a layout should be the layout editor as this is the least error prone way.

Changing the used layout can be done via Settings /Layout.

Layout Editor

You can start the layout editor via the Settings page , Layouts .

In this dialog you can select a file name for the layout you are going top modify. If currently a system layout was active (that you cannot change) a new user layout with the same name will be created and activated. If the layout already exists you can decide to overwrite it - or just choose a new name. You can still decide at the end of the editing process to discard your changes.

After starting the layout editor you will have a red border at AvNav - and you can only see pages where you can adapt the layout on.


By selecting a chart you can reach the navigation page as usual.

All widgets are located in so called "panels". Within a panel you cand drag and drop the widgets as long as the editor mode is active (but you cannot move them from one panel to another one this way).

On the navigation page you can assign different widgets to the panels depending on your display width. On a small display (value can be changed at Settings/small display) you can have different display then on a large one.

Via you can bring up a dialog for configuring the panels on a page.

The available panels are marked at the picture. Within the dialog you can select which panels should be visible - deepending on the display mode (small/default). If there is a check in the "small" column it means that you will have a different set up for the small mode. In the example above the top panel is not visible in normal mode, the left panel has a different content in normal and small mode but the bottom panels have the same content both in normal and small mode. At "current conditions" you can select if you would like to edit the panels for normal or small mode.

Both the bottom panels can have 2 rows of widgets (Setting: "2 widget rows") if there are widgets that do not fit any more the will be hidden completely.

Widget Dialog

After finishing the panel dialog you can now move the widgets within the panels or click on a widget or free space of a panel to bring up a widget dialog.

Depending on the kind of widget the dialog will contain different settings.

If you would like to move the widget to another panel you can select the target panel in the top most row of the dialog.

With "Update" you will store your changes.

At "New Widget" you can select a new widget from a list of available widgets.

Each widget needs an information about the data it should display. All displayable data is internally available in a store and each item is addressed by a key.

Depending on the type of widget the key(s) it will use to fetch the data from the store is fixed (i.e. built in) or has to be choosen in the dialog. The caption, unit and the css class can be set at most widgets. Assigning a css class is helpfull if you want to adapt the look and feel via css).

Formatter

Most of the widgets need a formatter to convert the data from the internal format into the display format. In most cases the formatter being used by a widget is fixed. But some of the allow some parameters (comma separated list) that you can assign in this dialog - e.g. you can change formatSpeed to convert to m/s instead of kn).

If you do not want to change the existing widget but add a new one - just use "Before" or "After" to insert it.

If there are more parameters you will see them in the dialog.

Dashboard Pages

You can configure up to 5 dashboard pages. On each of them you can have up to 5 panels.

On an empty dahsboard page you have to start with the panel configuration - .

You can have different display if currently anchor watch is active (in the default layout this is used on page 1).

If you have dahsboard pages without panels they will not appear in the display later on.

In principle you can also adpat the display on the route editing page (from the navigation page) but the display for the currently editing route and for the list of waypoints has to stay visible there.

When you are done with the layout editing you still need to save your layout using the button (red border) on each page.

At this point you can still decide to discard all of your changes.

When you save the layout it will immediately become active in your current browser (and will be stored at the server). Other browsers will only use the new layout if they reload the app or change the layout.

Layout Download/Upload

Via the Files/Download page , subpage you can download/upload/edit and delete the layout files. The currently active layout and system layouts can only be downloaded.

If you are deleting a layout that is currently used within another browser and this browser reloads the app it will write this layout back to the server - so it will re-apply.

Special Widgets

Despite a long list of predefined widgets (like SOG, COG, BRG, AisTargetWidget,...) where you can only change the caption, unit, formatterParameters and css class there are a couple of special widgets.

Default

This widget internally is the basis for most of the simple display widgets. But you also use it to define some displays in the layout editor.

In any case you need to select a value (the key for the store) and a formatter.

When clicking on value you will be prompted with a list of available data for display.


First of all those are the values predefined internally in AvNav. If you have enabled the signalk integration you will also see all received values from signalk (below vessels/self) with a prefix nav.gps.signalk. 

You have to take care to select a formatter that fits the value you selected.

Gauges

As already mentioned avnav integrates canvas-gauges. AvNav preconfigures a couple of widgets based on this library. Their parameters can be set in the layout editor.

There are linear gauges (linGauge...) or radial gauges (radGauge...).
Via the layout editor only a couple of the possible parameters can be set directly.

If you want more control or utilize different features of the canvas gauges library you should define own widgets as described in adapting with user specific java script code .

SignalK

The signalk plugin also provides a couple of widgets - they especially contain formatters for values that are otherwise not included in AvNav like signalKCelsius for displaying temperatures in °C.

They only make sense if you have appropriate signalk data, of course.

Adapting the Look and Feel of a Widget

If you would like to adapt the look and feel of a widget you should add an on css class to it. Afterwards you can assign rules to it in your user css.

Own Widgets

With user defined Java Script Code you can easily create your own widgets - both with simple HTML and with canvas based graphics.
