Converting Tracks to Routes

AvNav is permanently recording tracks. For every day a new track file (gpx) is written.

Those tracks (or tracks that have been uploaded at the Files/Download page) can be converted to routes to use them in navigation.

Concept

For this conversion you typically need to reduce the number of points in the track as they often have more then 1000 points. Otherwise use such a route is nearly impossible.

AvNav is using an algorithm that is taken from GPS Babel. One after the other the points will be removed that have the least significant distance from the line between the two neighboring points.

You can provide a parameter to define the maximum tolerable distance. The larger distance you allow the fewer points the created route will contain. You should try to keep your routes below 50 points to make them nicely usable.

After the conversion you can fine tune the route in the Route-Editor.

Handling

You can reach the conversion dialog either from the Files/Download page from within the Track Info Dialog (Convert Button) or from the info dialog if you have the track selected as an Overlay on the chart and click on a point within it. Again you have the  Convert Button - see picture.

After clicking "Convert" you will get the donversion dialog.

Normally (as in the picture) you will get a warning about your point caount being above 50.

Using "max Xte" you conmfigure the converter and start the conversion using "Compute". If the number of points still does not fit you can tray again with a changed xte.

In the example a xte of 22m gave me a good route.

Within the dialog you can still change the name for the new route (and decide to potentially overwrite an existing one with the same name).

After saving you can directly start editing the route.

At this point it counts if you had the track as an overlay at the chart as you can now easily check if all important points from the track are available in the route.
