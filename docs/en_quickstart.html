General

AvNav has been designed to be usable on touch devices (also with relatively small screens). The idea was to allow easy usage also under "on board" conditions.

You can handle AvNav also with mouse and keyboard, of course.

Installation and Set Up

AvNav is available in 2 variants:

  1. Client-Server
    The server will be installed onto a Linux or Windows system (like a Raspberry Pi). As "client" you will use an arbitrary browser to interact with the system (e.g. on a tablet or smart phone)
  2. Android App
    The complete functionality is bundled within the app. Additional devices can connect with a browser.

Client Server

AvNav is available as  package for various Linux distributions (Debian packages, Rpm) and as a Windows installer.
The debian packages are hosted within a repository und can be downloaded from the release page.
Additionally we maintain images for the Raspberry Pi. There is a detailed documentation available in the chapter installation.

After installing and starting up you can use your browser to connect to AvNav and you will get the WebApp.
If you are using our images the raspberry will create a Wifi network with an SSID of avnav..., passwors: avnav-secret. If you will connect to this network you can reach AvNav with the url http://avnav.avnav.de.
If you are connected to the server in a different way you can use http://avnav.local:8080 (does not work on android) - or you can use the IP of the server.
For IOS and Android devices I would recommend using a Bonjour Browser. Those tools can find the AvNav servers in the local network and will start a browsers without the need of typing an address.

Configuration

After installing AvNav (or when using a ready to go image) you can start normally without any additional configuration. AvNav will scan connected USB devices for serial adapters and will try to determine the necessary baud rate. Depending on the kind of installation a Udp receiver on port 34667 is active.

To adapt AvNav to your particular needs you can edit the config file avnav_server.xml. Additionally you can adapt the look and feel with settings, layout adaptation and with user defined code/CSS and plugins.

At Android all settings are directly integrated in the App (no avnav_server.xml).

NMEA Data

Client Server

AvNav is mainly handling NMEA0183 Data, from connected USB devices, serial ports, bluetooth devices,  TCP (client and server) or UDP. A NMEA multiplexer is integrated that allows for flexible configuration for the flow of received and send data.

A couple of NMEA sentences is decoded within AvNav (position daten, AIS,...) and used for it's display and routing functions.

In cooperation with Canboat and SignalK you can also handle NMEA2000 data. Additionally you can use all data from SignalK for the own ship to display them within AvNav.

The following NMEA sentences are decoded:

  • !AIVDM
  • $xxGGA
  • $xxGSV
  • $xxGLL
  • $xxVTG
  • $xxRMC
  • $xxMWV
  • $xxDPT
  • $xxDBT
  • $xxXDR (since 20210114)
  Depending on the configuration AvNav is able to create the following NMEA records:
  • $GPRMB
  • $GPAPB
  • $AVXDR
  • $AVMDA
  • $AVMTA

Android

At Android you can utilize the internal GPS. Additionally you can have a TCP or bluetooth connection for receiving GPS or AIS data. If your devices has USB-OTG available you can also connect an USB-serial adapter.

The following NMEA sentences will be decoded:

  • !AIVDM
  • $xxGGA
  • $xxGSV
  • $xxGLL
  • $xxGSA
  • $xxRMC
  • $xxMWV
  • $xxDBT
  • $xxXDR (since 20210114)
AvNav is able to send $GPRMC on an IP/USB/Bluetooth connection if configured.

Charts

AvNav is handling raster charts. You can download them from various sources in the net (e.g. OpenSeaMap or NOAA) - or you can use software like MobileAtlasCreator  or SASPlanet to download/create charts.

AvNav can directly handle charts in gemf and mbtiles formats. Additionally you can convert other types of raster charts like BSB (kap). This can be done directly on the server (raspberry) - or better beforehand on a desktop system.

Additionally AvNav can handle oesenc charts from o-charts (not on Android). The company is offering charts for Open Source Software at reasonable prices.

Charts have to be uploaded to AvNav before being usable - at the  Files/Download page, section charts . o-charts have to be uploaded via the o-charts plugin.

More details in the chapter charts.

Displays

At the Navigation page on top of the map there will be displays for the boat position, ist course, the route to the next waypoint, the current route , AIS targets and their courses, navigation circels and defined overlays.

At the Navigation page, inside the Route Editor and at up to 5 Dashboard pages you can display the values of all available navigational data. This includes the values from SignalK.

You can use simple numeric displays,analog gauges or graphical displays.

The displays can be adapted to your preferences. There is a Layout Editor and you can create / adapt displays with some lines of Java Script Code and CSS.

You can define different sets of displays and store the on the AvNav server. For each display device you can select the set ("layout") that you would like to use.

There is also an adaptation to different screen sizes being available in the layout ("small").

Routes

You can easily create and edit routes within AvNav. You will utilize the chart view at the Route Editor. Normally you just move the center of the chart (cross) to the next point you would like to add and with a button click you add this to the route. You can easily move, edit or delete points within the route.

Waypoints or other routes that are shown as overlays can be added to the route.

You can invert the route or empty it. Within AvNav routes are stored as gpx files. You can export or import them at the Files/Download page. From within the Route Editor you can start the route navigation immediately. Within a route there will be an alarming if you reach the next waypoint (if you are within an "approach" distance) and AvNav will automatically switch to the next waypoint.

Tracks

AvNav is constantly writing the current track and showing this on the chart. It tries to minimize the points zeichnet beständig den aktuellen Track auf. Dabei wird versucht, die Zahl der Trackpunkte zu begrenzen, indem nur bei größeren Änderungen oder nach einer bestimmten Zeit erneut ein Trackpunkt geschrieben wird (see Konfiguration ANVTrackWriter ). The tracks will be written as gpx file at regular intervals and can be exported an imported at the Files/Download page (you can also show their data like length and time at this page). Every day a separate gpx file is created. Available tracks can be shown as overlay on the chart.

You can  convert a track into a route. There is some logic available to reduce the number of points.

AIS

On your chart there will be a display of the AIS targets within a defined range (default: 20nm) with their positions and courses. There si a computation of the next target with a warning if a defined minimal distance is not met.

The symbols for the AIS targets can be replaced by own symbols - optionally different ones depending on the AIS shiptype. For the own boat and for each AIS target a course vector is drawn that will point to the position you will reach after a defined time interval (default: 10 minutes).

By clicking an AIS target (or the "next AIS" display) you will get all information for this target and you can change to the list of all AIS targets.

Alarm

AvNav create alarms for:

  • Anchor Watch
    On the Dashboard pages you can activate the anchor watch. Depending on the layout the displays will be adapted. If your boat leaves the defined circle or if the gps signal is lost an alarm will be created.
  • Approaching the next waypoint
  • Man over Board
    On all pages there is a separate Man-over-board button. By clicking this button the current position will become the routing target, all other routings will be stopped. Additionally the alarm is being created.

All alarms will be handled on the AvNav server. This way you can switch of all display devices but any alarm handling (like e.g. anchor watch) will continue to work.

You can create a sound (both at the server and at display devices) for the alarm. At the server you can trigger arbitrary commands whenever an alarm occurs (see example) . The configuration is handled within the configfile - AVNAlarmHandler.

Nightmode

At the Main page you can active the night mode . All displays will be adapted.

Adaptations

You can adapt AvNav to your personal needs with various technologies. As all the display is handled within a browser you can use CSS to customize your display.

The server configuration is done with the file avnav_server.xml.

Beside adapting the displays within the Layout editor you can easily create your own displays with some Java Script.

You can also include the display of other web pages (external ones or pages you created within AvNav). This will be handled as "User Apps".

The symbols that are used for various displays can be changed with a json file - like the  keybord short cuts.

With Python, Java Script and CSS you can write plugins.
