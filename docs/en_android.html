AvNav Android

After having the AvNav web GUI for the Raspberry Pi  already in place for some years there is now also a pure android version. It needs android starting from 4.4.(KitKat).

The most current version is available at downloads . Alternatively in the  Play Store.
Older versions or daily builds you can find via the  installation instructions.

Basically the app is packaging the WebApp (see User documentation) into an android app. For Charts see the hints at creating/converting charts.

The app can use the internal GPS of your device. Additionally you can receive NMEA data via TCP/IP, bluetooth or with an USB-serial adapter if your device supports USB OTG. As a sender for NMEA data you can e.g. use the raspberry version - or any other NMEA0183-wifi gateway. You can also use a simple bluetooth GPS mouse if your device does not have an internal GPS.

Charts are primarily expected in gemf Format. Since version 20200325 you can also use mbtiles or xml files for online chart sources. Charts can be located in 2 directories:

  • in the charts folder within the app working directory (choosen at start up). As this must be writable it will typically not be possible to put this on an SD card in newer android versions.
  • in a freely selectable other directory (also on an SD card) "additional charts dir" in the settings
    In the external charts dir you cannot directly use mbtiles. They must be copied into the working directory using the app .

When starting up there are some demo online charts being available.

If the choosen working directory is not yet there it will be created on first start up.(default: internal-sd-card/avnav).

The track and log data will be written into the track directory below the working directory.

In the routes sub folder the routes will be store in gpx format. Routes and tracks can be provided to other apps using the download function. To save them you need to have a file manager installed.

Bluetooth


To connect to a bluetooth device just pair it outside the app before starting AvNav.
Inside the app go to settings->bluetooth and select your device.

To leave the app use the button on the main page. This will shut down the background service that is writing the track even if the app is in background otherwise.

Modes

The app can run in 2 modes:
  • Normal
  • External Browser
In the normal mode the browser is integrated in the app and it behaves like a normal android app.
In external browser mode the app starts up a web server an a port that you can  select (default 34567) and shows the following page.


By clicking on "START BROWSER" you will be presented with a list of installed browsers and can select your favorite. I would recommend chrome.
Normally in the browser you will always have the address bar that wastes space. By using "add to home screen" in the settings you can put it on your start screen - and if you start from there it will run full screen.






To stop the app in this mode use the notification (sailboat) to go back to the app and leave it via "EXIT".
On the next start up only start the app and afterwards start your browser via the home screen icon.

If you select Settings>Android->MainSettings "WebServer external access" other devices in the same network can access your app.

Take care: Never use this in a publci network as there is no protection inside the app. Any user in the same network can access your device!
The app will annouce itself via bonjour as avnav-android so other devices can easily find it.

When the app is running the track data will be written to a file every 5 minutes.
Important Hint:
If you start up your browser once from the app and once from the home screen you will end up with different settings in the app.
