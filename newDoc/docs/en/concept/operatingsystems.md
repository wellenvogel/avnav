# Operating Systems
Thanks to this architecture, AvNav can be run on a wide variety of devices. Before you begin the installation, you face the decision of which platform is best suited for your individual needs. Generally, there are three typical scenarios.

## Raspi OS 
The solution preferred by many users is installing it on a Raspberry Pi—a small, inexpensive Linux mini-computer. This scenario is ideal for boat owners who want a dedicated, reliable system. The Raspberry Pi is configured specifically for AvNav and permanently installed below deck. For the display in the cockpit, you then simply use one or more standard tablets that connect wirelessly to this central computer. If you want to use AvNav as a permanent, central navigation solution, this is the recommended path.

## Android
A very popular alternative is using AvNav as an Android app. This is the quickest and most straightforward way to get started. If you own an Android tablet, you can install AvNav on it and use the tablet as a completely independent ("standalone") system. The app then uses the tablet's built-in GPS. This solution is excellent for smaller boats, charter sailors who want to bring their own system, or simply as a backup system in a drawer. By the way, the server concept applies here as well: the Android tablet can simultaneously serve as the server for other devices on board.

## Windows or Linux
Finally, there is also the option to install AvNav on a standard PC or laptop running Windows or Linux. This is particularly practical if you want to plan your trips, organize waypoints, or generate new charts on a large screen at home during the winter. Later, the finished plans can easily be transferred to the onboard system. Such a setup is also excellent as a fallback solution on a laptop at the chart table.

No matter which path you choose, the user interface and operation in the browser always look exactly the same. So you do not have to adapt or relearn anything if you switch from the Android version to a Raspberry Pi system.
